## Planning document

This is the planning document for an idea I am trying to create. The purpose for this application is so that a person can track, analyze, and be provided information about waste/consumption habits and best ways to use them. How this will be done is by breaking down resource consumption by the habits that they usually have.

### How the person will interact with the app(v1)

For the V1 implmentation I want the app to have the basic quality of providing user recommendations for food plans they can import into whatever site/application they want. This means that the application will have set directions from the user about their eating habits such as if they are vegetarian, if they are trying to lose/gain weight, high protein, keto, etc... This customization from the user will provide the application context on how to move forward. Then after all the information is processed, the user will be provided a meal plan with locations nearby that sell the given items. This way the user has instant and easy knowledge about where, when, and what to by from the store. After the purchase the user can take a picture of the recipt and it will process the items the user purchased to store in a database on the device. The user will be provided metrics about the food they purchased such as calories, protein, carbs, fat, etc... as well as the resource consumption these food products took to be manufactured. Alltogether, the workflow is able to help users bring their eating habits to be sustainable as well as yummy.

### How the application will look

The application will look clean and easy to use. It will be a mobile app capable of running on multiple devices such as IOS and android. This implementation will be powerful as people from everywhere will be able create a network. I want the design principles to be clean so for that reason we will be using shadcn_flutter for the design UI.
